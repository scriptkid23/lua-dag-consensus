# Design: Milestone B — L3 Full part A (03c-2a)

**Date:** 2026-05-22
**Status:** Draft (awaiting review)
**Audience:** Contributors implementing the macro-proposer + backup + Mode B leaderless fallback
**Relations:** Extends [`2026-05-22-l3-macro-finality-design.md`](2026-05-22-l3-macro-finality-design.md) (03c-1 minimal slice, now landed). Does **not** include Mode A subnet rotation, `byzantine_split` real adversary, EWMA reputation driver, beacon chaining, real ECVRF proofs, real BLS sign/verify, or slashing — those are 03c-2b (Mode A + adversary scenarios), 03d (slashing + real BLS), and a coordinated ECVRF upgrade with bullshark anchor.

---

## 1. Goals

Replace round-robin proposer selection with whitepaper-faithful sortition and add the backup-proposer + leaderless fallback chain so a missing primary no longer stalls a macro window. Land the remaining L3 protocol pieces below Mode A:

- **VRF sortition** for proposer selection, mirroring `bullshark::select_anchor` (publicly-derivable `beta_i`, no real ECVRF proofs yet).
- **Shoal reputation** plumbed into the score formula but held neutral (`1.0`) — the EWMA driver defers to 03c-2b.
- **Backup proposer + `T_macropropose` timer** chain.
- **Mode B leaderless fallback** when both primary and backup miss their slots.
- The existing `apps/sim::scenarios::mode_b_fallback` stub lit up with a real adversary that suppresses primary + backup proposals; assertion: end-to-end finality through Mode B.

**Non-goals (03c-2a):** Mode A subnet rotation, real ECVRF with VRF proofs, EWMA reputation driver, beacon chaining (`Eq. 8.1`), real BLS sign/verify, real `byzantine_split` adversary, `network_partition` L3 assertion extensions, slashing.

## 3. Architectural decision: `TimerScheduler` hoisted to `StateMachine`

Both bullshark and macro_fin need to allocate `TimerId`s. To prevent collisions (each book owns a `HashMap<u64, _>` and silently reacts only to its own ids), the allocator must be shared. Cleanest: hoist `TimerScheduler` from `WaveBook` to `StateMachine`. Three bullshark entry points (`on_certified_vertex`, `on_timer_fired`, the internal `try_emit_for_wave`) gain a `&mut TimerScheduler` parameter; one allocator call site (`waves.timers.allocate()`) becomes `timers.allocate()`. New macro_fin entry point (`on_timer_fired`) and existing `on_local_micro_qcs` + `on_macro_proposal` get the same parameter. The `WaveBook.timers` field is removed.

**Rejected alternatives:**

- **Namespace by starting offset** (macro starts at `u64::MAX/2`): hacky, brittle if either allocator ever wraps or is changed.
- **Module discriminant on `TimerId`**: wire-breaking change to `TimerId(u64)`; affects Borsh encoding of events / persistence; cost too high for the benefit.

This is the only signature break in 03c-2a, and it stays inside `consensus` — no call site outside the crate touches `TimerScheduler` directly.

## 5. Phase 03c-2a — protocol behavior

### 5.1 Purpose

Prove the **VRF sortition → primary timeout → backup → backup timeout → Mode B leaderless → MacroQc** flow on deterministic sim. Keep `consensus` pure; reuse all existing Action/Event/port surfaces; add no new wire types.

### 5.2 Algorithm specification (explicit)

Let `n` = validator count, `f = (n-1)/3`, `W = cfg.macro_fin.macro_window_w = 8`, `T = cfg.timing.t_macropropose_ms`.

| Whitepaper feature | 03c-2a behavior |
|--------------------|-----------------|
| Proposer sortition | `beta_i = blake3_with_dst(BEACON, beacon ‖ height.to_be_bytes() ‖ validator_id)`. Score = `vrf_sortition_score(beta_i, total_stake, own_stake, reputation = 1.0)`. **Primary** = min score across the set; **backup** = second-min. Ties: smaller `ValidatorId.as_bytes()` wins (matches bullshark anchor convention). `MacroProposal.vrf_proof = VrfProof::zero()` (real ECVRF deferred). |
| Shoal reputation | Plumbed into `vrf_sortition_score` at the call site but every validator passes `reputation = 1.0`. The EWMA driver updating reputation on observed proposer behavior defers to 03c-2b. Unit tests use non-neutral fixtures to pin the formula. |
| Backup proposer | Backup is computed alongside primary in `ProposerSchedule::vrf_sortition`. On `T_primary` fire (no `MacroProposalReceived` for `height` arrived), backup self-elects: if `self_id == schedule.backup`, emit `BroadcastMacroProposal(local_candidate)`; else schedule `T_backup`. |
| `T_macropropose` timer | Delay = `cfg.timing.t_macropropose_ms * 1_000_000` nanoseconds (default 4 s = 16 sim rounds at 250 ms/round). Scheduled by every non-primary on candidate build. Scheduled by every non-backup on `T_primary` fire. Cancelled on `MacroProposalReceived` adoption via `Action::CancelTimer`. |
| Mode B leaderless | On `T_backup` fire, validator transitions `mode_by_height[height] → ModeBLeaderless`, then: if `lock_macro.try_lock(self_id, height, candidate.hash) == Ok`, emit `BroadcastBlsPartial { subnet: SubnetId(0), validator: self_id, checkpoint_hash: candidate.hash, sig: fixture }` and `UpdateBlobStatus(SoftConfirmed)`. The candidate is the locally-built `MacroBook.candidate[height]` from `on_local_micro_qcs`. |
| `MacroProposalReceived` validation | Accept proposer ∈ `{schedule.primary, schedule.backup}` (3c-1 accepted only primary). Parent + micro_root + canonical-hash checks unchanged from 03c-1. Cancels pending timer for `height` if any. |
| `MacroQc.mode` label | Read from `mode_by_height[height]` when aggregating. Mapping: `MacroMode::Mode0Flat → AggregationMode::Mode0Flat`, `MacroMode::ModeBLeaderless → AggregationMode::ModeBLeaderless`, `MacroMode::Pending` or missing entry → `AggregationMode::Mode0Flat` (defensive fallback; `on_bls_partial` never reaches QC build before mode is set on the happy path, so the fallback is a safety net pinned by a unit test). Local label, not consensus property — two validators may produce MacroQcs with same `checkpoint_hash` but different `mode` (one flipped to Mode B before/after the other got a proposal). |
| `on_macro_qc_received` (late-arrival path) | Unchanged from 03c-1. Idempotent merge regardless of incoming `MacroQc.mode`. |
| Beacon | Static (sim `VirtualBeacon` initial value). Per-height rotation comes from `height` byte in `beta_i`. Chaining deferred. |
| Lock-macro suppression | `lock_macro.try_lock` failure increments `suppressed_conflicts` (stat only) in both `on_macro_proposal` and `handle_backup_timeout`. No `EmitSlashEvidence` (slashing → 03d). |

### 5.3 `MacroBook` extensions

Three new fields appended to the existing `MacroBook` struct:

```rust
/// Per-height aggregation mode tracker.
/// Pending → Mode0Flat on first valid MacroProposalReceived adoption.
/// Pending → ModeBLeaderless on T_backup fire (after both primary + backup missed).
pub(crate) mode_by_height: HashMap<Height, MacroMode>,

/// Per-active-timer ownership. Maps allocator-issued `TimerId.0` →
/// the macro height + timer kind it represents.
pub(crate) macro_timers: HashMap<u64, (Height, MacroTimerKind)>,

/// Reverse lookup: per-height current outstanding timer (for the cancel path).
pub(crate) pending_timer_by_height: HashMap<Height, TimerId>,
```

Two new enums (in `macro_fin/book.rs`):

```rust
#[derive(Clone, Copy, Debug, Eq, PartialEq)]
pub enum MacroMode {
    Pending,
    Mode0Flat,
    ModeBLeaderless,
}

#[derive(Clone, Copy, Debug, Eq, PartialEq)]
pub enum MacroTimerKind {
    Primary,   // T_macropropose for primary slot; on fire, backup takes over
    Backup,    // T_macropropose for backup slot; on fire, Mode B activates
}
```

### 5.4 `StateMachine` struct + dispatch changes

```rust
pub struct StateMachine {
    cfg: Config,
    emitted: EmittedSet,
    waves: WaveBook,
    macros: MacroBook,
    /// Shared timer ID allocator. Bullshark and macro_fin both allocate from
    /// here; each module tracks ownership via its own (TimerId → context) map.
    timers: TimerScheduler,
}
```

Bullshark refactor: `WaveBook.timers` field deleted. Three bullshark entry points gain a `&mut TimerScheduler` parameter. The internal helper `try_emit_for_wave` receives it too. The one allocator call inside `try_emit_for_wave` becomes `timers.allocate()`. No algorithmic change in bullshark.

`step` dispatch updates:

```rust
match event {
    Event::CertifiedVertexReceived(cv) => {
        let mut actions = crate::bullshark::on_certified_vertex(
            &mut self.emitted, &mut self.waves, &mut self.timers, &self.cfg, cv, ctx,
        )?;
        crate::macro_fin::on_local_micro_qcs(
            &mut self.macros, &mut self.timers, &self.cfg, ctx, &mut actions,
        )?;
        Ok(actions)
    }
    Event::MicroQcAssembled(qc) => crate::bullshark::on_micro_qc_assembled(&self.emitted, qc),
    Event::TimerFired(id) => {
        let mut actions = crate::bullshark::on_timer_fired(
            &mut self.emitted, &mut self.waves, &mut self.timers, &self.cfg, id, ctx,
        )?;
        crate::macro_fin::on_timer_fired(
            &mut self.macros, &mut self.timers, &self.cfg, ctx, id, &mut actions,
        )?;
        Ok(actions)
    }
    Event::MacroProposalReceived(p) =>
        crate::macro_fin::on_macro_proposal(
            &mut self.macros, &mut self.timers, &self.cfg, p, ctx,
        ),
    Event::BlsPartialReceived(bp) =>
        crate::macro_fin::on_bls_partial(&mut self.macros, &self.cfg, bp, ctx),
    Event::SubnetAggregateReceived(_) => Ok(Actions::new()),
    Event::MacroQcReceived(qc) =>
        crate::macro_fin::on_macro_qc_received(&mut self.macros, qc, ctx),
    Event::ValidatorSetUpdated { .. } => Ok(Actions::new()),
    Event::SlashEvidenceFound(_) => Ok(Actions::new()),
}
```

Both bullshark and macro_fin see every `TimerFired`; each silently returns when the id isn't in its ownership map. Because the allocator is shared, ids cannot collide.

### 5.5 Data flow (end-state 03c-2a)

```text
                  StateMachine::step
                          │
                          ▼ Event::CertifiedVertexReceived(cv)
            bullshark::on_certified_vertex   (allocates wave timers via shared scheduler)
                          │
              Actions: [..., BroadcastMicroQc(qc)?]
                          │
                          ▼
         macro_fin::on_local_micro_qcs
         ├─ micro_ring.push(qc.checkpoint_hash)
         ├─ if ring.len() == W:
         │     micro_root  = H(MACRO_MICRO_ROOT ‖ concat(W))
         │     candidate   = build(h+1, micro_root, last_macro_hash, epoch)
         │     ring.clear()
         │     schedule    = vrf_sortition(set, beacon, height)
         │     mode_by_height[height] = MacroMode::Pending
         │     if self_id == schedule.primary:
         │         emit BroadcastMacroProposal(local candidate, proposer = self, ZERO_VRF, fixture sig)
         │     else:
         │         id = sm.timers.allocate()
         │         macro_timers[id.0] = (height, Primary)
         │         pending_timer_by_height[height] = id
         │         emit ScheduleTimer { id, delay = T_macropropose }

                          ▼ Event::MacroProposalReceived(p)
         macro_fin::on_macro_proposal
         ├─ schedule = vrf_sortition(set, beacon, p.checkpoint.height)
         ├─ p.proposer ∈ {primary, backup}?                       else drop
         ├─ p.checkpoint.parent == last_macro_hash?               else drop
         ├─ p.checkpoint matches local candidate?                 else drop
         ├─ cancel pending timer (if any):
         │     emit CancelTimer(id); macro_timers.remove(&id.0); pending_timer_by_height.remove(&height)
         ├─ try_lock(self_id, height, p.checkpoint.hash):
         │     Err  → suppressed_conflicts += 1, empty
         │     Ok   →
         │       mode_by_height[height] = MacroMode::Mode0Flat
         │       two_chain.adopt; candidate.insert; partials[hash].insert(self_id)
         │       emit BroadcastBlsPartial{ SubnetId(0), self, hash, fixture }
         │       emit UpdateBlobStatus(SoftConfirmed)

                          ▼ Event::TimerFired(id)
         bullshark::on_timer_fired (existing — only reacts if id ∈ wave timers)
         macro_fin::on_timer_fired
         ├─ Some((height, kind)) = macro_timers.remove(&id.0)?   // not ours → empty
         ├─ pending_timer_by_height.remove(&height)
         ├─ schedule = vrf_sortition(set, beacon, height)
         ├─ match kind:
         │     Primary:
         │       if self_id == schedule.backup:
         │         emit BroadcastMacroProposal(local candidate, proposer = self, ZERO_VRF, fixture sig)
         │       else:
         │         next_id = sm.timers.allocate()
         │         macro_timers[next_id.0] = (height, Backup)
         │         pending_timer_by_height[height] = next_id
         │         emit ScheduleTimer { next_id, delay = T_macropropose }
         │     Backup:
         │       mode_by_height[height] = MacroMode::ModeBLeaderless
         │       candidate = book.candidate[height]
         │       if try_lock(self_id, height, candidate.hash) == Ok:
         │         partials[candidate.hash].insert(self_id)
         │         emit BroadcastBlsPartial{ SubnetId(0), self, candidate.hash, fixture }
         │         emit UpdateBlobStatus(SoftConfirmed)
         │       else:
         │         suppressed_conflicts += 1

                          ▼ Event::BlsPartialReceived(bp)
         macro_fin::on_bls_partial (unchanged from 03c-1, with one line change)
         ├─ ... existing logic ...
         ├─ if 2f+1 reached:
         │     qc.mode = mode_by_height.get(&height).copied().unwrap_or(MacroMode::Mode0Flat)
         │            .into()    // MacroMode → types::macros::AggregationMode
         │     emit BroadcastMacroQc, PersistMacroCheckpoint, PersistMacroQc,
         │          UpdateBlobStatus(Justified), optional UpdateBlobStatus(Finalized)
```

### 5.6 New entry point: `macro_fin::on_timer_fired`

```rust
pub fn on_timer_fired(
    book: &mut MacroBook,
    timers: &mut TimerScheduler,
    cfg: &Config,
    ctx: &HostContext<'_>,
    id: TimerId,
    actions: &mut Actions,
) -> Result<()> {
    let Some((height, kind)) = book.macro_timers.remove(&id.0) else {
        return Ok(()); // not a macro timer; bullshark already handled it (or it's stale)
    };
    book.pending_timer_by_height.remove(&height);

    let set = ctx.valset.set_for(Epoch(0))?
        .ok_or_else(|| crate::Error::InvalidConfig("no validator set".into()))?;
    let beacon = ctx.beacon.current()?;
    let schedule = ProposerSchedule::vrf_sortition(&set, &beacon, height);

    match kind {
        MacroTimerKind::Primary =>
            handle_primary_timeout(book, timers, cfg, &schedule, height, actions),
        MacroTimerKind::Backup =>
            handle_backup_timeout(book, &schedule, height, actions),
    }
}
```

Both helpers are private free functions inside `macro_fin/mod.rs`. They implement the timer-chain logic from §5.5.

### 5.7 Sim driver changes (03c-2a)

**`apps/sim/src/world.rs`:**

- New field: `macro_proposal_drop_count: u32` (target drops per height).
- New field: `macro_proposals_dropped_by_height: HashMap<Height, u32>` (running counter).
- New method: `pub fn set_macro_proposal_drop_count(&mut self, n: u32)`.
- In `apply_actions`, intercept `Action::BroadcastMacroProposal(p)`:

  ```rust
  Action::BroadcastMacroProposal(p) => {
      let h = p.checkpoint.height;
      let dropped = self.macro_proposals_dropped_by_height.entry(h).or_insert(0);
      if *dropped < self.macro_proposal_drop_count {
          *dropped += 1;
      } else {
          self.net.enqueue_from_action(
              validator_idx,
              &Action::BroadcastMacroProposal(p),
              n, now, &mut self.rng,
          );
      }
  }
  ```

  The drop is at the *send* side — the proposer's local SM state is untouched (it doesn't receive its own proposal back). The counter is per-height so each height's primary and backup are independently silenced.

**`apps/sim/src/scenarios/mode_b_fallback.rs`** (replace stub):

```rust
const DEFAULT_ROUNDS: u32 = 160;

#[must_use]
pub fn run(validators: u32, rounds: u32, seed: [u8; 32]) -> Report {
    let cfg = Config::default_table_17_1();
    let rounds = rounds.max(DEFAULT_ROUNDS);
    let mut world = World::new(validators, seed, cfg.clone());
    world.set_network_conditions(NetworkConditions::with_round_jitter(
        cfg.timing.round_duration_ms,
    ));
    world.set_macro_proposal_drop_count(2); // suppress primary + backup → force Mode B
    world.run(rounds);
    Report {
        scenario: "mode_b_fallback".into(),
        validators,
        rounds,
        safety_ok: checker::safety::check(&world).is_ok(),
        liveness_ok: checker::liveness::check(&world).is_ok(),
        lock_macro_ok: checker::lock_macro::check(&world).is_ok(),
        notes: vec![
            "mode_b_active_for_every_height".into(),
            "network_jitter_from_round_duration".into(),
        ],
    }
}
```

**`apps/sim/src/checker/safety.rs`** — macro block deleted entirely. The 03c-1 check ("all MacroQcs with same `checkpoint_hash` are byte-identical") becomes false under 03c-2a because `MacroQc.mode` and `agg.bitmap` are per-validator local labels (one validator's MacroQc carries `mode=Mode0Flat`, another's same-hash MacroQc carries `mode=ModeBLeaderless` after a local Mode B flip). Fork detection moves entirely to `lock_macro` checker — it groups by `Persistence::macro_checkpoint_at(height)` and verifies all validators adopted the same `(height, MacroCheckpoint.hash)`, which is consensus-determined regardless of QC mode. The MicroQc block in `safety.rs` is unchanged (`MicroQc.mode` doesn't exist; MicroQc equality on same `checkpoint_hash` still holds). A doc comment at the deleted-block site points to `lock_macro` checker as the canonical macro-fork detector.

**Default rounds for `mode_b_fallback`:** 160 = 5 macro windows × 32 rounds/window. Each window needs ≥ 2 × T_macropropose = 32 rounds to walk through primary timeout → backup timeout → Mode B activation. With 32 rounds per macro window and 32 rounds of timer chain per window, 5 windows fits comfortably in 160 rounds.

- `apps/sim happy_path --rounds 96`: `safety_ok && liveness_ok && lock_macro_ok` (regression — proposer flow now via VRF sortition, no round-robin).

## 8. Error handling

- `step` continues to return `Err` only for invariant violations (no validator set for `Epoch(0)`, malformed timer state). All honest factory inputs in 03c-2a paths return `Ok`.
- `macro_fin::on_macro_proposal` silently drops on: wrong proposer (not primary or backup), parent mismatch, micro_root mismatch, local candidate mismatch — same as 03c-1. These are honest-disagreement conditions; surfacing them as errors would mask the real failure (timeout → backup → Mode B).
- `macro_fin::on_timer_fired` silently returns `Ok(())` when `id ∉ macro_timers` (the id belongs to bullshark or is stale after cancellation).
- Mode B partial emission failure due to `lock_macro.try_lock` Err: counted in `suppressed_conflicts`, no panic, no error.

## 10. Risks and mitigations

| Risk | Mitigation |
|------|------------|
| `TimerScheduler` hoist breaks bullshark | One-field move + threading `&mut TimerScheduler` through 3 entry points; one allocator call site swap; existing 7 bullshark tests pin behavior. Diff is mechanical. |
| Macro timer fires after Mode B already activated (race with late `BlsPartial`) | `macro_timers.remove(&id.0)` is atomic; if `mode_by_height[height]` is already `ModeBLeaderless`, `handle_backup_timeout` short-circuits before re-locking. Re-lock would also be idempotent (same hash). |
| `MacroProposalReceived` arrives after `T_backup` already activated Mode B locally | `lock_macro.try_lock` returns `Ok` because the proposal hash matches the locked hash (every honest validator built the same candidate). Re-emitted partial is harmless duplicate; sim safety checker tolerates same-hash partials from the same validator. |
| Per-validator `MacroQc.mode` divergence breaks existing safety checker | Tightened to compare only `checkpoint_hash` (§5.7); doc comment explains why; `lock_macro` checker (per-height via `Persistence::macro_checkpoint_at`) remains the strict fork-detection check. |
| `t_macropropose_ms = 4000` (16 sim rounds) eats too much sim time for `mode_b_fallback` | Scenario defaults to 160 rounds (5 macro windows × 32 rounds); 2 × T_macropropose per window = 32 rounds, fits 1:1 per window with no idle time but fits. If flaky, raise to 192. |
| Sim drop counter off-by-one if primary and backup happen to be the same validator | `vrf_sortition` constructor guarantees primary ≠ backup unless `n == 1`. Asserted via `debug_assert!` in the constructor. 4-validator sim always has distinct primary/backup. |
| Mode B suppresses proposal but candidate `parent` mismatch | Mode B chain is self-consistent: every honest validator builds the same candidate chain because they all see the same MicroQc sequence. After Mode B finalizes height `h`, every honest validator's `last_macro_hash = candidate[h].hash`. Next candidate's `parent` still matches. |
| `Actions` SmallVec cap (16) overflow | Per-step max in 03c-2a: `TimerFired` → 2 (BroadcastMacroProposal OR BroadcastBlsPartial + UpdateBlobStatus); other events unchanged from 03c-1 (max 5). Cap holds comfortably. |
| Mode-tracker default fallback `unwrap_or(Mode0Flat)` masks a missing-height bug | Acceptable in 03c-2a: every code path that reaches `on_bls_partial` after either successful proposal adoption or Mode B activation sets `mode_by_height` first. The `unwrap_or` is a safety net, not a primary path; unit test pins both branches. |
