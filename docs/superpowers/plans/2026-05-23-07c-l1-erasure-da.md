# L1 Erasure Coding + DA Skeleton (07c) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace sequential raw chunks with Reed–Solomon erasure shards, upgrade `BlobRef.commitment` to an RS merkle root, populate `ChunkRef` in the store, implement causal-set RPC for late joiners, and add a challenge verify hook skeleton — without full DAS, KZG, or on-chain slashing.

**Architecture:** `crates/dag/erasure/` implements RS encode/decode over blob bytes with fixed `(k, n)` from config. Publish path encodes payload → stores **n shards** (not sequential slices). Gossip wire gains optional erasure metadata on `BlobChunk`. `BlobRef.commitment` becomes `blake3_with_dst(BLOB_RS_ROOT, merkle(shard_commitments))`. `crates/net/src/rpc/causal_set.rs` gets a live handler backed by `LiveDag`. Challenge structs live in `crates/dag/da/` with verify hooks only (no slash emission).

**Tech Stack:** Rust 1.88, `reed-solomon-erasure` (or equivalent GF(2^8) RS crate), `borsh`, `blake3`, `types`, `storage`, `net` RPC, `apps/node`.

**Spec:** [`docs/superpowers/specs/2026-05-23-l1-availability-dag-design.md`](../specs/2026-05-23-l1-availability-dag-design.md) §6 (Phase C).

**Prerequisite:** 07b landed (`BlobStore`, `blob-chunk` gossip, `BlobRef` on vertices, `lua_submitBlob`).

## Current gap

| Area | Today (post-07b) | Target |
|------|------------------|--------|
| Chunks | Sequential raw slices | RS data + parity shards |
| `ChunkRef` | Wire type only | Stored + gossiped with shard index |
| Commitment | `BLOB_COMMIT(full_payload)` | RS merkle root (`BLOB_RS_ROOT`) |
| Recovery | Must hold all sequential chunks | Reconstruct from any **k** shards |
| Sync | None | `CausalSetReq` RPC returns vertex hashes |
| DA enforcement | None | Challenge struct + verify hook (no slash) |

## Design decisions (lock-in)

| Topic | Decision |
|-------|----------|
| RS parameters | Default `k = 4`, `n = 6` (4 data + 2 parity); config `[node].erasure_k`, `erasure_n` |
| Shard size | `data_shard_size_bytes = 32768` (32 KiB per data shard); pad payload to `k * shard_size` |
| Shard index | `0..k-1` data shards, `k..n-1` parity shards; maps to `ChunkRef.index` |
| Wire (07c) | Extend `BlobChunk` with `shard_kind: ErasureShard` enum `{ Data, Parity }` — backward compat: 07b chunks treated as `Sequential` behind flag |
| Store | Same `blob_chunk` CF; value includes erasure metadata borsh prefix |
| Availability (07c) | Blob available when **any** `k` distinct shards present **and** decode succeeds |
| Commitment | Merkle tree over per-shard `blake3_with_dst(BLOB_SHARD, shard_bytes)`; root hashed with `BLOB_RS_ROOT` |
| KZG | **Out of scope** — `BlobRef.commitment` stays `Hash32` opaque |
| Causal-set RPC | `lua_getCausalSet(from, to)` → `CausalSetResp { hashes }` from `LiveDag` index |
| Challenges | `AvailabilityChallenge { blob_id, chunk_indices, responder }` + `verify_challenge_response` hook returning `Ok/Err` only |
| SM / slash | **No** `EmitSlashEvidence` from DA hook in 07c |
| Config gate | `[node].l1_erasure_enabled = true` devnet; `false` keeps 07b sequential path |

## Done — 07c acceptance criteria

- Payload publish produces **n** RS shards; blob recoverable from any **k** shards.
- `BlobRef.commitment` matches RS merkle root when erasure enabled.
- `lua_listBlobChunks` returns populated `ChunkRef` entries.
- `lua_getCausalSet` returns certified vertex hashes for a round range.
- `verify_availability_response` accepts honest responses and rejects tampered shards.
- 07b sequential path still works when `l1_erasure_enabled = false`.

**Non-goals (explicit):**

- 2D RS / full DAS sampling
- KZG polynomial commitments
- On-chain slashing / `EmitSlashEvidence` from DA
- Peer sampling protocol (validators randomly challenge on wire)
- Cross-validator shard repair / re-encode

**Next:** Devnet E2E with rollup submitting erasure-coded blobs; future `crates/dag/das/` for 2D RS + KZG per folder-arch §10.
